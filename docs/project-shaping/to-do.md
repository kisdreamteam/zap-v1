# Project Shaping Framework V2

Purpose:

Design the product before building the product.

Goal:

Reduce ambiguity, scope creep, overengineering, rework, and implementation drift.

# Constant Rules (Always Active)

* [ ] Documentation before implementation
* [ ] Product decisions before architecture decisions
* [ ] Simplicity over complexity
* [ ] Scope control over feature growth
* [ ] Validation before expansion
* [ ] Update documentation before changing direction
* [ ] Avoid future-proofing unless justified
* [ ] Smaller solutions win
* [ ] Build only what is needed now
* [ ] Revisit assumptions regularly
